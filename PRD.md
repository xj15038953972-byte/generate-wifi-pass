## 1. 产品概述

### 1.1 背景

家庭与小型办公场景中，用户设置 WiFi 密码时常见两类问题：

1. **密码太弱**：使用生日、手机号、简单数字序列，易被暴力破解或字典攻击。
2. **密码难记难传**：随机强密码虽然安全，但口头告知访客或手动输入时体验差。

本产品是一个 **纯前端、无需注册** 的 WiFi 强密码生成工具网站，帮助用户一键生成符合 WPA/WPA2/WPA3 规范的强密码，并提供复制、可选 QR 码分享等能力。

### 1.2 产品定位

- **类型**：单页工具站（Utility Web App）
- **核心价值**：安全、快速、零学习成本
- **差异化**：专注 WiFi 场景（长度限制、字符集建议、易读模式），而非通用密码生成器

### 1.3 产品目标

| 目标 | 说明 |
|------|------|
| 安全 | 使用密码学安全的随机源（`crypto.getRandomValues`） |
| 易用 | 3 秒内完成生成并复制 |
| 轻量 | 首屏加载 < 2s（静态部署） |
| 隐私 | 密码仅在浏览器本地生成，不上传服务器 |
| 可部署 | 支持 GitHub Pages / Vercel / Netlify 静态托管 |

## 4. 功能需求

### 4.1 核心功能（MVP / P0）

#### F1 — 密码生成

- **随机模式**（默认）：
  - 长度可调：**8–63** 字符（WPA-PSK 上限 63）
  - 默认长度：**16**
  - 字符集开关：
    - 大写字母 A–Z（默认开）
    - 小写字母 a–z（默认开）
    - 数字 0–9（默认开）
    - 符号 `!@#$%^&*()-_=+[]{}` 等（默认开）
  - 至少勾选一种字符集；若组合无法生成（极端情况），给出提示
- **随机源**：必须使用 `window.crypto.getRandomValues`，禁止 `Math.random()`
- **操作**：「重新生成」按钮；修改任意选项后自动重新生成（可配置）

#### F2 — 强度评估

- 基于长度、字符集多样性、常见弱密码模式给出评分：
  - 等级：弱 / 中 / 强 / 极强
  - 颜色指示：红 / 黄 / 绿 / 深绿
- 少于 8 位时显示警告（不符合 WPA 最低建议）
- 纯数字或纯字母时提示增强字符类型

#### F3 — 复制到剪贴板

- 一键复制，成功后 Toast 提示「已复制」
- 复制失败时降级：选中密码文本 + 手动复制提示

#### F4 — 默认推荐预设

- 提供「推荐设置」快捷按钮：
  - 长度 16
  - 四类字符全开
  - 排除易混淆字符（见 P1，MVP 可先内置为默认开启）

### 4.2 增强功能（P1）

#### F5 — 排除易混淆字符

- 开关：排除 `0/O/o`、`1/l/I/i` 等
- 默认：开启

#### F6 — WiFi QR 码

- 输入项（可选，有默认值）：
  - SSID（网络名称）
  - 加密类型：WPA/WPA2（默认）、WEP（不推荐，需警告）、nopass（开放网络，需警告）
- 生成标准 WiFi QR 格式：`WIFI:T:WPA;S:{ssid};P:{password};;`
- 支持下载 PNG
- **注意**：QR 码仅在客户端生成，不上传

#### F7 — 生成历史（本地）

- 使用 `sessionStorage` 保存本次会话最近 5 条（可选）
- 提供「清除历史」按钮
- 不做持久化云端存储

### 4.3 未来功能（P2）

#### F8 — Passphrase 模式

- 从词表随机抽取 4–6 个词，用 `-` 连接
- 示例：`correct-horse-battery-staple-42`
- 显示等效熵估算

#### F9 — 多语言

- 中文（默认）、英文
- i18n 键值结构，便于扩展

#### F10 — PWA

- 可安装到主屏幕
- 离线可用（Service Worker 缓存静态资源）

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| 首次内容绘制（FCP） | < 1.5s |
| 可交互时间（TTI） | < 2s |
| 密码生成耗时 | < 50ms |
| 包体积（gzip） | < 100KB（MVP，不含 QR 库时 < 50KB） |

### 5.2 安全与隐私

- **零服务端**：MVP 阶段纯静态站点，无后端 API
- **无遥测默认**：不默认收集密码；若后续加 Analytics，不得记录密码字段
- **无第三方依赖处理密码**：密码不经过任何外部服务
- **CSP**：部署时配置 Content-Security-Policy
- **HTTPS**：生产环境强制 HTTPS

### 5.3 兼容性

| 平台 | 最低版本 |
|------|----------|
| Chrome / Edge | 最近 2 个 major |
| Firefox | 最近 2 个 major |
| Safari | 15+ |
| 移动端 | iOS Safari 15+、Android Chrome |

### 5.4 无障碍（a11y）

- 所有控件可键盘操作
- 密码区域有 `aria-label`
- 强度指示不仅依赖颜色，还有文字标签
- 对比度符合 WCAG AA

### 5.5 SEO

- 页面 title / description / og tags
- 语义化 HTML（`h1` 唯一）
- 目标关键词：`wifi password generator`、`strong wifi password`、`generate wifi password`

## 7. UI/UX 要求

### 7.1 设计原则

- **极简**：首屏即见密码，设置项次级
- **可信**：安全说明可见但不打扰（Footer 或折叠面板）

- 等宽字体展示密码（如 `JetBrains Mono`、`ui-monospace`）

### 7.3 关键交互

| 交互 | 行为 |

## 12. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 被误解为破解工具 | 法律/品牌风险 | 文案明确「仅生成新密码」 |
| `Math.random` 误用 | 密码可预测 | Code review + 单元测试强制 crypto API |
| QR 码泄露密码 | 物理暴露 | 提示用户勿在公开场所展示含密码 QR |
| 字符集含路由器不支持的符号 | 无法连接 | 提供「兼容模式」预设（仅字母数字） |
| GitHub Pages 缓存 | 更新延迟 | 文件名 hash + 文档说明 |
